# 阿鑫 IP

## 角色定义

阿鑫是这套个人 IP 配图流程的固定视觉角色。

阿鑫不是吉祥物、贴纸或头像，而是个人内容工坊里的真人手绘操作员。他负责把原始经验、真实项目、判断、证据、复盘和输出，转换成可以公开发布、反复引用、跨平台复用的内容资产。

## 外形

- 真人手绘男生形象，黑色蓬松短发，圆框眼镜，白色或浅色 hoodie，背后或衣服上可出现“阿鑫 / Axin”字样。
- 线稿以黑色为主，头发可以用密一点的黑色排线；五官简洁，表情安静、专注、轻微冷幽默，不卖萌。
- 身体比例自然但保持手绘草图感，不做写实肖像，也不要二次元夸张脸。
- 手上经常拿橙色线缆、文件、标签、插头、清单或工具，动作要真的推动流程。
- 画面里可以出现小红色警示牌、红色标记或蓝色 agent 图标，用于表达风险、系统反馈或 AI 协作。
- 允许变换站姿、侧脸、半身、背影、坐姿，但不要改变黑发、眼镜、hoodie 这三个识别点。

## 性格

- 很认真，但做的事有点荒诞。
- 像一个低调的内容系统操作员。
- 冷幽默，不卖萌。
- 有时显得手忙脚乱，但判断清醒。
- 重视证据、复用和真实输出，不喜欢空喊口号。
- 对“看起来完成但其实没跑通”的东西非常敏感。

## 常见职责

让阿鑫承担核心动作：

- 把真实项目接入内容机器。
- 给未经验证的观点贴红色警示标签。
- 用橙色线缆连接素材、判断、发布口。
- 检查证据、文件、README、复盘和发布入口。
- 把混乱素材整理成内容卡片。
- 在发布闸门前做质检。
- 从旧项目里回收可复用零件。
- 把长文拆成短内容卡片。
- 把一次真实运行记录整理成公开案例。
- 把中文内容和英文入口接到同一套资产流水线。
- 检查 README、llms.txt、docs、平台快照和示例图是否一致。

## 禁止

- 不要把阿鑫画成可爱吉祥物。
- 不要画成儿童卡通角色。
- 不要把阿鑫画成非人类工具、印章、黑盒、机器人、动物、表情包或抽象符号。
- 不要改掉黑发、眼镜、hoodie 的主识别点。
- 不要给阿鑫复杂服装、夸张表情或闪亮眼睛。
- 不要让阿鑫只是站在角落里看。
- 不要让阿鑫抢走结构表达。
- 不要把阿鑫画成小黑、Bongo Cat、已有海绵类动画角色或其他受保护角色的变体。
- 不要让图只剩角色展示而没有认知动作。

## 判断标准

如果去掉阿鑫，图的核心隐喻还能完全成立，说明阿鑫太装饰了；要重写提示词，让阿鑫成为动作主体。
