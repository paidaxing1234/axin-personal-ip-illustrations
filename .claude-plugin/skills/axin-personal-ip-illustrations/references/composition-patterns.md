# 构图模式与原创规则

## 基础结构类型

选择一种结构即可，不要混太多。

### 内容工坊

适合：把经验、记录、素材加工成文章、短视频脚本、README、案例。

画法：左侧输入，中间阿鑫操作低科技工位，右侧输出资产块。橙色箭头表达主流向。

### 证据质检

适合：强调真实运行、可验证结果、不要 mock、不要空喊。

画法：阿鑫在发布闸门前称重、盖章、贴红色警示。通过的内容进发布口，未通过的回到素材箱。

### 前后对比

适合：混乱/有序，临时/可复用，手动/自动，焦虑/稳定。

画法：左边混乱，右边稳定，中间由阿鑫操作一个转换动作。

### 资产分层

适合：个人 IP 的能力栈、内容资产层、信任层、产品层。

画法：一层层盒子，不要正式金字塔。阿鑫在旁边搬砖、贴标签或拧螺丝。

### 发布路径

适合：从项目到内容，从内容到 GitHub，从 GitHub 到社媒承接。

画法：一条弯曲路径，少量节点，阿鑫用橙色线缆连接节点。

### 复利飞轮

适合：一次项目复用成多篇内容、多张图、多份文档、多次引流。

画法：一个不太圆的手绘飞轮，阿鑫在旁边上油、压印或捡掉落资产。

### 角色状态

适合：创作者卡住、工具太多、素材混乱、终于跑起来。

画法：2-4 个小状态，每个状态只保留一个动作和一个短标注。

### 小漫画分镜

适合：失败到成功、真实过程、吐槽、避坑。

画法：2-4 个小场景，每格只表达一个动作，不要做成长教程。

## 原创隐喻生成法

每次都从当前内容重新发明隐喻，不能照搬旧图。

### 三步

1. 把抽象概念换成物理动作：压印、称重、过筛、接线、装箱、回收、开闸、贴警示、发货。
2. 把系统结构换成低科技物件：印章、旧工位、质检台、纸箱、秤、闸门、线缆、抽屉、回收箱、手摇机器。
3. 让阿鑫承担动作：不是站旁边，而是压、称、接、推、贴、捞、拧、守、拆、记录。

## 可用物件池

- 印章、压印台、旧工位、纸箱、抽屉、秤、闸门、线缆、漏斗、回收箱、打孔器、邮筒、发布口、质检灯。
- 每张图只选 1-2 个主物件，不要堆满。

## 反复刻规则

示例图片只用于校准线条密度、留白、颜色克制和阿鑫参与方式。

不要默认复刻这些构图：

- 左输入、中间机器、右输出的标准流水线，除非主题确实需要。
- 阿鑫站在旁边看机器。
- 一张图里塞满所有平台和渠道。
- 只画头像或角色展示。
- 大量箭头和节点的正式流程图。

同类主题也要换新隐喻。例如“内容复利”不一定画飞轮，可以画阿鑫把一块经验印泥压出多种印章；“信任建立”不一定画桥，可以画阿鑫给证据块称重后逐块堆成台阶。
