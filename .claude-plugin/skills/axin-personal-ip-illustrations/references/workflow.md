# 个人 IP 配图生产流程

## 输入

可以输入：

- 文章草稿。
- 项目复盘。
- README 或开源项目介绍。
- 视频脚本。
- 公众号、知乎、小红书内容提纲。
- 一个单独观点。
- 一个真实运行结果或截图。

## 提炼

先问这几个问题：

- 这段内容要让读者记住什么判断。
- 哪个地方最能体现“我真的做过”。
- 哪个地方需要从抽象概念变成物理动作。
- 哪个地方可以沉淀成资产，而不是一次性表达。

## 选图

优先选择：

- 核心判断。
- 输入输出闭环。
- 前后对比。
- 证据质检。
- 资产复用。
- 发布承接。
- 常见坑。
- 一次项目如何变成多种内容。

不要平均配图。不要为了数量而配图。

## 生成

每张图都先写短 brief：

- 主题。
- 核心意思。
- 结构类型。
- 阿鑫动作。
- 主物件。
- 中文短标注。

然后套用 `prompt-template.md`，优先使用 `image_gen` 单张生成；如果当前环境只暴露其他生图工具，再使用对应工具。不要把多张图拼成一张。

## 抖音和小红书图文

如果输入目标是抖音图文、小红书图文、竖屏轮播或类似短内容卡片，默认从正文配图切换到 9:16 竖屏轮播：

- 默认尺寸：9:16，优先 `1080x1920` 或当前 `image_gen` 支持的等比例竖图。
- 默认数量：5-8 张，包含封面、痛点、流程、分支、复用和行动页。
- 默认工具：只要环境暴露 `image_gen`，就直接逐张调用 `image_gen`。
- 每张图只讲一个观点，不要做信息过密的海报。
- 中文标题要短，复杂正文留给平台排版，不强迫生图模型生成大段中文。
- 保存到 `assets/<topic-slug>-douyin-carousel/`。

## 保存

正文配图默认保存到：

```text
assets/<topic-slug>-illustrations/
```

命名格式：

```text
01-core-judgement.png
02-evidence-check.png
03-asset-loop.png
```

角色锚点和风格样片保存到：

```text
axin-personal-ip-illustrations/assets/examples/
```

## 发布

一组完整交付建议包含：

- 原文或主题。
- shot list。
- 每张图的 prompt。
- 最终 PNG。
- QA 结论。
- 可复用标签：主题、用途、结构类型、是否适合公开仓库 README。
