# 阿鑫角色资产库规划

## 目标

角色资产库用于减少每次生图时的漂移。阿鑫的识别点固定为：黑发、圆框眼镜、浅色 hoodie、真人手绘、安静专注、白底黑线稿、少量红橙蓝。

任何新图都应该先对照主视觉和角色锚点，再扩展动作和场景。

## 建议补齐的角色角度

1. 正面半身：用于文章开头、工具介绍、头像附近的轻量场景。
2. 侧面接线：用于流程分发、内容管线、自动化。
3. 背影操作台：用于复盘、发布、真实项目转内容。
4. 坐姿审稿：用于代码审查、文档审查、证据核验。
5. 低头整理文件：用于案例库、素材回收、内容复利。
6. 举红色警示牌：用于风险、未验证、不要 mock。
7. 和蓝色 agent 信号协作：用于 llms.txt、GEO、AI 工作流。

## 动作库

- 接线：连接真实项目、中文内容、英文入口和 agent。
- 核验：检查文件、截图、提交、脚本和运行输出。
- 分流：把同一套资产拆成公众号、复盘、README.en、llms.txt。
- 贴警示：标记未验证、过时、名称不一致和假完成。
- 整理：把散乱记录变成案例卡片和发布包。
- 回收：从旧项目里捡可复用资产。

## 禁止漂移

- 不要变成非人类角色、机器人、工具箱、印章或抽象 logo。
- 不要变成过度可爱、儿童卡通或表情包。
- 不要丢掉黑发、眼镜、hoodie。
- 不要让阿鑫只是站在角落里看。
- 不要使用平台 logo 作为主要视觉元素。
